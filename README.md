# Анализ методов массового поиска подстрок: сравнительное исследование CPU и GPU реализаций

**Автор:** Артемов Т.М.  
**Группа:** 6133-010402D

---

## Аннотация

В рамках лабораторной работы решалась задача множественного поиска подстрок в сплошном буфере байтовой последовательности. Рассмотрены три подхода: эталонный алгоритм прямого перебора, выполняемый на центральном процессоре (CPU); параллельная реализация с распределением подстрок между потоками графического процессора (GPU); а также матричный метод, предложенный в методических указаниях, основанный на однопроходной обработке буфера с атомарным обновлением состояния.

---

## 1. Реализация на центральном процессоре (CPU-эталон)

Алгоритм представляет собой трёхуровневую вложенную итерацию:

- внешний цикл — перебор всех подстрок из заданного множества;
- средний цикл — последовательный обход всех позиций буфера;
- внутренний цикл — посимвольное сравнение текущей подстроки с фрагментом буфера.

Условием обнаружения подстроки служит полное совпадение всех её символов с соответствующими байтами буфера.

Для тестовых данных (буфер объёмом 100 Кбайт, 2000 подстрок переменной длины от 3 до 15 байт) время выполнения составило **829.97 мс**, что принято за базовое значение для оценки ускорения.

---

## 2. Параллельная реализация на GPU с декомпозицией по подстрокам (GPU-A)

В данном варианте на GPU создаётся пул потоков, каждый из которых отвечает за поиск одной подстроки. Внутри потока сохраняется итеративный обход буфера, аналогичный CPU-версии, однако все такие обходы выполняются одновременно благодаря массовому параллелизму графического процессора.

Достигнутое ускорение составило приблизительно **5.33×**: время сократилось до **155.78 мс** относительно эталонного CPU-алгоритма.

---

## 3. Матричный метод на GPU (однопроходный, GPU-B)

Данный подход, описанный в третьем разделе методического пособия, использует иную модель вычислений. Основные этапы:

1. **Формирование матрицы** размерностью `|S| × |B|`, где `|S|` — количество подстрок, `|B|` — размер буфера. Начальное значение каждой ячейки равно длине соответствующей подстроки.

2. **Однократный проход по буферу** с обработкой каждого байта отдельным потоком GPU. Если буфер содержит 100 000 байт, запускается 100 000 потоков.

3. **Атомарное обновление** — каждый поток идентифицирует подстроки, в которых встречается обрабатываемый символ, и уменьшает значения соответствующих ячеек матрицы.

4. **Критерий обнаружения** — подстрока считается найденной, если соответствующая ячейка (в конце обработки) достигла нуля.

Ключевое отличие от предыдущих методов — отсутствие многократного сканирования буфера. Вместо того чтобы для каждой подстроки выполнять полный обход данных, производится единый проход, в ходе которого информация обновляется сразу для всего множества подстрок.

**Результат:** время выполнения — **10.13 мс**. Ускорение относительно CPU-эталона — **81.90×**, относительно простого GPU-метода (GPU-A) — **15.38×**.

## Верификация

Оба GPU-метода (Kernel-A и Kernel-B) успешно прошли верификацию относительно эталонного CPU-алгоритма. Результаты поиска полностью совпадают во всех трёх реализациях.

---

## Вывод

Матричный метод демонстрирует существенное преимущество за счёт редукции вычислительной сложности: вместо многократного сканирования буфера для каждой подстроки выполняется лишь один проход с параллельной обработкой символов и атомарными обновлениями. Это позволяет эффективно использовать ресурсы GPU даже при большом количестве подстрок. На целевой платформе (NVIDIA Tesla T4) достигнуто ускорение более чем в **80 раз** по сравнению с последовательной CPU-реализацией.
